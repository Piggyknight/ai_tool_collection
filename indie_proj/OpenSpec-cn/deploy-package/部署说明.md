# OpenSpec-cn v1.4.0 团队部署包

## 📦 部署包说明

本部署包包含 OpenSpec-cn v1.4.0 的完整功能，无需网络连接即可安装和使用。

## 🚀 快速部署步骤

### 1. 解压部署包

```bash
# 解压到目标位置
tar -xzf openspec-cn-1.4.0.tar.gz

# 或者直接复制整个文件夹
cp -r openspec-cn-1.4.0 /path/to/your/team/tools/
```

### 2. 测试安装

```bash
# 进入部署包目录
cd openspec-cn-1.4.0

# 验证安装
node bin/openspec.js --version
# 应该输出: 1.4.0

# 测试新功能
node bin/openspec.js redmine --help
node bin/openspec.js sprint --help
```

### 3. 配置环境

#### 方案A: 直接使用 (推荐团队新手)
```bash
# 每次使用时进入目录
cd /path/to/openspec-cn-1.4.0
node bin/openspec.js redmine setup
```

#### 方案B: 创建快捷命令 (推荐资深用户)
```bash
# 添加到系统PATH
export PATH="/path/to/openspec-cn-1.4.0/bin:$PATH"

# 创建别名
alias openspec='node /path/to/openspec-cn-1.4.0/bin/openspec.js'

# 现在可以直接使用
openspec redmine setup
openspec sprint create my-sprint
```

#### 方案C: 安装到项目 (推荐开发团队)
```bash
# 在你的项目中
cd your-project
npm install /path/to/openspec-cn-1.4.0

# 在package.json中添加脚本
{
  "scripts": {
    "review": "openspec-cn review change",
    "sprint": "openspec-cn sprint",
    "redmine": "openspec-cn redmine"
  }
}

# 使用
npm run redmine setup
npm run sprint list
```

## 📋 使用示例

### 团队协作工作流

**开发者A**: Sprint规划
```bash
openspec sprint create sprint-2026-Q2
openspec sprint plan sprint-2026-Q2
```

**开发者B**: Story开发
```bash
openspec story breakdown sprint-2026-Q2 user-auth
openspec task breakdown user-auth
openspec task sync user-auth  # 开发过程中同步进度
```

**开发者C**: 代码审查
```bash
openspec review change user-auth --sync  # 审查并同步到Redmine
```

**项目经理**: 总结归档
```bash
openspec sprint-spec generate sprint-2026-Q2
openspec sprint close sprint-2026-Q2
```

## 🎁 团队部署建议

### 推荐部署位置
- **团队共享服务器**: `/opt/openspec-cn/`
- **个人开发机**: `~/tools/openspec-cn/`
- **CI/CD服务器**: `/usr/local/openspec-cn/`

### 配置管理
```bash
# 创建团队配置模板
cat > ~/.openspec/config.yaml << EOF
redmine:
  enabled: true
  cli-path: /usr/local/bin/red-cli
  instances:
    - name: dev
      server: https://redmine-dev.example.com
      api-key: YOUR_DEV_KEY
      project-id: 1
EOF
```

### 权限设置
```bash
# 设置适当的权限
chmod 755 /path/to/openspec-cn-1.4.0/bin/openspec.js
chmod 644 /path/to/openspec-cn-1.4.0/package.json
```
